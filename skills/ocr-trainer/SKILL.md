# OCR Trainer Skill

This skill helps you train a custom OCR model (EasyOCR/ClovaAI) to recognize text in your specific game font (e.g., Mabinogi).

## Prerequisites

1.  **Python Environment:** Ensure Python 3.8+ is installed.
2.  **GPU (Highly Recommended):** Training on CPU is extremely slow. Use a machine with NVIDIA GPU + CUDA.
3.  **Dependencies:**
    ```bash
    pip install fire lmdb opencv-python nltk natsort torch torchvision
    ```

## Workflow

### 1. Generate Training Data
You should already have synthetic data generated by `backend/generate_training_data.py`.
- **Location:** `backend/train_data/`
- **Structure:**
  - `images/`: Contains `.png` images.
  - `labels/`: Contains `.txt` files with the text content.

### 2. Create LMDB Dataset
EasyOCR requires data in LMDB format. Use the provided script to convert your images/labels.

```bash
python3 skills/ocr-trainer/scripts/create_lmdb_dataset.py \
  --input backend/train_data \
  --output backend/train_data_lmdb
```

### 3. Setup Training Repository
Clone the official deep-text-recognition-benchmark repository (which EasyOCR is based on).

```bash
git clone https://github.com/ClovaAI/deep-text-recognition-benchmark.git
cd deep-text-recognition-benchmark
```

### 4. Run Training
Run the training command. This will train a model to recognize your specific font.

**Key Parameters:**
- `--train_data`: Path to your LMDB folder (`backend/train_data_lmdb`).
- `--valid_data`: Use the same path or a separate validation set.
- `--select_data /`: Use root folder.
- `--batch_ratio 1.0`: Use all your data.
- `--Transformation None`: Game text is usually rigid, so strict transformation isn't always needed, but `TPS` is good for robust OCR.
- `--FeatureExtraction ResNet`: Standard backbone.
- `--SequenceModeling BiLSTM`: Standard sequence modeling.
- `--Prediction CTC`: Connectionist Temporal Classification (fast and good for this).

**Command:**
```bash
python3 train.py \
    --train_data ../backend/train_data_lmdb \
    --valid_data ../backend/train_data_lmdb \
    --select_data / \
    --batch_ratio 1.0 \
    --Transformation TPS --FeatureExtraction ResNet --SequenceModeling BiLSTM --Prediction CTC \
    --saved_model "" \
    --num_iter 5000 \
    --valInterval 500 \
    --batch_size 64
```

*Note: Adjust `batch_size` based on your VRAM. Increase `num_iter` (e.g., to 50000) for better results.*

### 5. Use Custom Model
Once training is complete, you will get a `.pth` file (e.g., `best_accuracy.pth`).

To use this in your **FastAPI Backend**:

1.  Move the `.pth` file to `backend/models/custom_mabinogi.pth`.
2.  Update `backend/main.py`:

```python
# Initialize Reader with custom model
reader = easyocr.Reader(
    ['ko', 'en'], 
    model_storage_directory='backend/models',
    user_network_directory='backend/models',
    recog_network='custom_mabinogi' # Name of your pth file without extension
)
```

## Troubleshooting
- **Out of Memory:** Reduce `--batch_size`.
- **Low Accuracy:** Check `train_data` visualization. Ensure labels match images. Increase `num_iter`.
